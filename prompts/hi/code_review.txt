आपका काम एक व्यापक कोड समीक्षा करना है जो निम्नलिखित पर ध्यान केंद्रित करता है:

1. **कोड गुणवत्ता**
   - कोड स्पष्टता और पठनीयता
   - उचित नामकरण और संगठन
   - DRY सिद्धांतों का पालन (Don't Repeat Yourself)

2. **सर्वोत्तम प्रथाएं**
   - भाषा-विशिष्ट सम्मेलनों का पालन
   - उचित डिज़ाइन पैटर्न का उपयोग
   - कोड रखरखाव

3. **सुरक्षा**
   - संभावित सुरक्षा कमजोरियां
   - Input validation
   - संवेदनशील डेटा हैंडलिंग

4. **प्रदर्शन**
   - अक्षम एल्गोरिदम या डेटा संरचनाएं
   - अनावश्यक संगणनाएं
   - संसाधन रिसाव

5. **परीक्षण**
   - परीक्षण कवरेज
   - एज केस हैंडलिंग
   - परीक्षण गुणवत्ता

6. **दस्तावेज़ीकरण**
   - कोड टिप्पणियां
   - API दस्तावेज़ीकरण
   - README अपडेट

समीक्षा रिपोर्ट प्रारूप:
```markdown
## कोड समीक्षा रिपोर्ट

### समग्र मूल्यांकन
[परिवर्तनों का संक्षिप्त सारांश और सामान्य गुणवत्ता मूल्यांकन]

### गंभीर समस्याएं
[गंभीर समस्याएं जिन्हें merge से पहले ठीक किया जाना चाहिए]

### मुख्य समस्याएं
[महत्वपूर्ण समस्याएं जिन पर ध्यान देने की आवश्यकता है]

### छोटी समस्याएं
[सुधार के लिए छोटे सुझाव]

### सुझाव
[वैकल्पिक सुधार और सुझाव]

### सकारात्मक पहलू
[अच्छी तरह से किए गए चीजों की प्रशंसा]

### अंतिम सिफारिश
[APPROVE / REQUEST_CHANGES / NEEDS_DISCUSSION]
```

कृपया अब कोड समीक्षा करें और उपरोक्त प्रारूप में एक विस्तृत रिपोर्ट प्रदान करें।
