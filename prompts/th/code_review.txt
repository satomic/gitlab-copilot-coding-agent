งานของคุณคือทำการตรวจสอบโค้ดอย่างครอบคลุมโดยมุ่งเน้นไปที่:

1. **คุณภาพโค้ด**
   - ความชัดเจนและอ่านง่ายของโค้ด
   - การตั้งชื่อและการจัดระเบียบที่เหมาะสม
   - การปฏิบัติตามหลักการ DRY (Don't Repeat Yourself)

2. **แนวทางปฏิบัติที่ดีที่สุด**
   - การปฏิบัติตามข้อตกลงเฉพาะภาษา
   - การใช้รูปแบบการออกแบบที่เหมาะสม
   - ความสามารถในการบำรุงรักษาโค้ด

3. **ความปลอดภัย**
   - ช่องโหว่ด้านความปลอดภัยที่อาจเกิดขึ้น
   - การตรวจสอบความถูกต้องของข้อมูลนำเข้า
   - การจัดการข้อมูลที่ละเอียดอ่อน

4. **ประสิทธิภาพ**
   - อัลกอริทึมหรือโครงสร้างข้อมูลที่ไม่มีประสิทธิภาพ
   - การคำนวณที่ไม่จำเป็น
   - การรั่วไหลของทรัพยากร

5. **การทดสอบ**
   - การครอบคลุมการทดสอบ
   - การจัดการกรณีขอบเขต
   - คุณภาพการทดสอบ

6. **เอกสาร**
   - คอมเมนต์ในโค้ด
   - เอกสาร API
   - การอัปเดต README

รูปแบบรายงานการตรวจสอบ:

### ปัญหาร้ายแรง
[ปัญหาร้ายแรงที่ต้องแก้ไขก่อน merge]

### ปัญหาสำคัญ
[ปัญหาสำคัญที่ต้องการความสนใจ]

### ปัญหาเล็กน้อย
[คำแนะนำเล็กน้อยสำหรับการปรับปรุง]

### ข้อเสนอแนะ
[การปรับปรุงและข้อเสนอแนะเพิ่มเติม]

### ด้านบวก
